# Entity Relationship (ER) Diagram

## Overview

The Entity Relationship (ER) Diagram represents the database structure of the **Loan Approval Prediction System**. It illustrates the core entities, their attributes, primary keys, and relationships required to manage applicant information, credit history, machine learning models, and loan approval predictions.

The database is designed to efficiently organize applicant data, process loan requests, and store prediction results generated by the machine learning model.

---

## Entities

The system consists of the following five primary entities:

- Users
- Applicant_Details
- Credit_History
- ML_Model
- Approval_Prediction

---

## Primary Keys

| Entity | Primary Key |
|---------|-------------|
| Users | `UserID` |
| Applicant_Details | `ApplicantID` |
| Credit_History | `HistoryID` |
| ML_Model | `ModelID` |
| Approval_Prediction | `PredictionID` |

---

## Relationships

- **Users → Applicant_Details (1:N)**
  - One user can manage or submit multiple applicant records.

- **Applicant_Details → Credit_History (1:N)**
  - One applicant can have multiple credit history records.

- **Applicant_Details → Approval_Prediction (1:1)**
  - Each applicant receives one final loan approval prediction.

- **ML_Model → Approval_Prediction (1:N)**
  - One machine learning model can generate multiple loan approval predictions.

- **Credit_History → Approval_Prediction**
  - Credit history information is used as an important input for generating loan approval predictions.

---

## Users Entity Attributes

| Attribute | Description |
|-----------|-------------|
| `UserID` | Primary Key |
| `Name` | User's full name |
| `Email` | User's email address |
| `Password` | Encrypted password |
| `Role` | User role (Admin/User) |

---

## Applicant_Details Entity Attributes

| Attribute | Description |
|-----------|-------------|
| `ApplicantID` | Primary Key |
| `UserID` | Foreign Key |
| `IncomeType` | Applicant income category |
| `EducationType` | Education level |
| `FamilyStatus` | Marital/family status |
| `HousingType` | Housing ownership status |
| `EmploymentDays` | Employment duration |

---

## Cardinality

The relationships between the entities are defined as follows:

- One **User** can manage multiple **Applicant_Details** records.
- One **Applicant** can have multiple **Credit_History** records.
- One **Applicant** receives one **Approval_Prediction**.
- One **Machine Learning Model** can generate predictions for multiple applicants.
- **Credit_History** data is used as an input for the loan approval prediction process.

---

## Database Design Summary

The ER diagram provides a structured and normalized database design for the Loan Approval Prediction System. It establishes relationships among users, applicants, credit history, machine learning models, and prediction results. This design minimizes data redundancy, maintains data integrity, and supports efficient storage, retrieval, and prediction of loan approval outcomes.
