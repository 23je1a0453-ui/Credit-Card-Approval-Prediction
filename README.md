# Running the Flask Application

## Overview

Running the Flask application is the final step in the **Credit Card Approval Prediction** project. This step verifies that the frontend, backend, trained machine learning model, and prediction logic are correctly integrated and functioning as expected.

The application is executed locally using the Flask development server. Once the server starts successfully, users can access the application through a web browser, submit applicant information, and receive real-time credit card approval predictions.

---

## Step 1: Open Anaconda Prompt

Launch **Anaconda Prompt** (or Command Prompt) and navigate to the project directory containing the `app.py` file.

```bash
cd path/to/Credit-Card-Approval-Prediction
```

---

## Step 2: Run the Flask Application

Start the Flask development server by executing the following command:

```bash
python app.py
```

> **Note:** Ensure that all required Python packages are installed before running the application.

---

## Step 3: Access the Application

After the server starts successfully, Flask displays a local URL similar to:

```text
http://127.0.0.1:5000/
```

Open this URL in any web browser to launch the application.

---

## Step 4: Navigate Through the Application

The application consists of three pages:

### Home Page (`home.html`)

- Displays an introduction to the Credit Card Approval Prediction system.
- Provides a button to navigate to the prediction page.

### Prediction Page (`index.html`)

- Allows users to enter applicant information.
- Collects financial and demographic details.
- Submits the data to the Flask backend.

### Result Page (`result.html`)

- Displays the prediction generated by the machine learning model.
- Shows whether the credit card application is **Approved** or **Not Approved**.

---

## Prediction Workflow

1. Open the application in a web browser.
2. Navigate to the prediction page.
3. Enter applicant details.
4. Click the **Predict** button.
5. Flask receives and processes the input.
6. The trained machine learning model generates a prediction.
7. The prediction result is displayed on the **Result** page.

---

## Benefits

- Verifies successful integration of the frontend and backend.
- Confirms the trained model is loaded correctly.
- Generates real-time predictions.
- Provides a simple and interactive user interface.
- Ensures the complete application functions as expected.
