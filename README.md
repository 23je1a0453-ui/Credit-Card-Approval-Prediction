# Flask Web Application

## Overview

The **Credit Card Approval Prediction** application includes a Flask-based web interface that allows users to enter applicant details and receive real-time credit card approval predictions. Flask serves as the bridge between the frontend interface and the trained machine learning model, providing a simple and interactive way to access the prediction system.

The application consists of three HTML pages stored inside the `templates/` directory and a backend Python script (`app.py`) that manages routing, model loading, input processing, prediction generation, and result display.

---

## Frontend Pages

### Home Page (`home.html`)

The **Home** page serves as the landing page of the application. It provides a brief introduction to the Credit Card Approval Prediction system and includes a navigation button that directs users to the prediction page.

**Features:**
- Project introduction
- User-friendly interface
- Navigation to the prediction form

---

### Prediction Page (`index.html`)

The **Prediction** page contains a form where users enter applicant information required for prediction.

Typical input fields include:

- Gender
- Age
- Income Type
- Education Level
- Employment Status
- Housing Type
- Annual Income
- Family Status
- Number of Family Members
- Other applicant details

After entering the required information, the user clicks the **Predict** button to submit the form.

---

### Result Page (`result.html`)

The **Result** page displays the prediction generated by the machine learning model.

Possible outputs include:

- **Approved**
- **Not Approved**

The prediction result is presented in a clear and user-friendly format.

---

# Backend Development

## Flask Application (`app.py`)

The `app.py` file serves as the main backend of the application.

Its responsibilities include:

- Initializing the Flask application.
- Loading the trained machine learning model.
- Defining application routes.
- Receiving user input from the HTML form.
- Processing input values.
- Generating predictions.
- Displaying the prediction result.

---

## Load the Trained Model

The trained model is loaded using **Pickle** or **Joblib**.

Example:

```python
import pickle

model = pickle.load(open("model.pkl", "rb"))
```

---

## Flask Routes

The application defines the following routes:

### Home Route

```python
@app.route("/")
def home():
    return render_template("home.html")
```

Displays the application's landing page.

---

### Prediction Route

```python
@app.route("/predict", methods=["POST"])
def predict():
```

**Purpose:**

- Receives user input from the form.
- Converts input values into the required format.
- Sends the processed data to the trained model.
- Generates the prediction.
- Displays the result on the Result page.

---

## Prediction Workflow

1. User opens the Home page.
2. Clicks **Predict**.
3. Enters applicant details.
4. Submits the prediction form.
5. Flask receives the input.
6. The trained machine learning model processes the data.
7. The prediction is generated.
8. The Result page displays whether the application is **Approved** or **Not Approved**.

## Summary

The Flask web application provides an intuitive interface for predicting credit card approval status. The frontend consists of **home.html**, **index.html**, and **result.html**, while the backend (`app.py`) handles routing, model loading, input processing, and prediction generation. By integrating Flask with the trained machine learning model, the application delivers fast, reliable, and real-time approval predictions through a web-based interface.
